# Problem 10: Insert OOXML directly (the escape hatch)

**Tier:** 3
**Concepts:** `Range.insertOoxml()`, when the typed API isn't expressive enough
**Prerequisites:** Problems 1–9
**Time budget:** ~50 minutes

## Statement

Add a button labelled `Insert styled block`. When clicked, it inserts at the end of the document a paragraph containing the text `Kata note` styled bold red 14pt, followed immediately by a second paragraph containing the text `— generated by kata` in italic gray. Use `Range.insertOoxml()` (or `Body.insertOoxml()`) with a hand-authored OOXML string.

The point of this problem is to *feel* the escape hatch. Some rich formatting combinations can't be expressed via the typed Office.js API cleanly (or the code is ugly). OOXML insertion lets you drop straight to Word's underlying XML format for full expressiveness.

## Constraints

- Must use `insertOoxml(ooxmlString, 'End')` (or `'After'` on a range).
- The OOXML string must be a well-formed Word document fragment (root `<pkg:package>` element wrapping the necessary parts, or a simpler `<w:document>` fragment depending on which API variant you use — check docs).
- Result must be observable in Word: bold red 14pt for the first paragraph, italic gray for the second.

## Automated verification

The test file at `tests/10-insert-ooxml.test.ts` asserts:

- The OOXML string references the expected style properties (bold `w:b`, italic `w:i`, colour attributes).

## Hints

<details>
<summary>Hint 1 — what to look up first</summary>
Microsoft Learn: "Range.insertOoxml method" and the "Understanding the Office Open XML file format" primer. You don't need to become an OOXML expert; you need the shape of a two-paragraph fragment with styling.
</details>

<details>
<summary>Hint 2 — common mistake</summary>
Hand-authoring OOXML from scratch. Instead: create a formatted document in Word, save as `.docx`, unzip, look at `document.xml` to see the exact OOXML Word emits for what you want. Copy the relevant `<w:p>` elements into your string.
</details>

<details>
<summary>Hint 3 — sketch of approach</summary>
Create the target formatting in a scratch Word doc. Save as `.docx`, unzip, extract the `<w:p>...</w:p>` pair for the two paragraphs. Wrap them in the `<pkg:package>` envelope Office.js expects. Insert via `body.insertOoxml(xml, 'End')`.
</details>
